# KBO 카드뉴스 주제 선정 정책

## 목적

이 문서는 자동화 watcher와 ranker가 사용할 초기 선정 정책이다.

목표는 완벽한 자동 선별이 아니라, 사람이 놓치면 아까운 주제를 빠르게 발견하고 Discord로 확인 요청을 보내는 것이다.

초기 정책은 과거 결과물 기반의 가산점 규칙으로 사용한다. 과거에 만들지 않았던 유형이라고 자동 탈락시키지 않는다.

## 분석 기준 데이터

분석 대상:

- `outputs/completed_topic_registry.json`
- `outputs/approval_run_*/03_title_html_editor_no_multimodal/title_html_editor_report.json`
- `outputs/approval_run_*/title_render_pngs/*.state.json`
- `outputs/approval_run_*/01_topic_candidates/topic_selection_choice.json`

현재 로컬 데이터 기준:

- 후보 선택 파일: 10개
- 후보 topic: 100개
- 선택 topic: 29개
- editor report topic: 29개
- completed registry topic: 21개
- render state: 31개

주의:

- 같은 topic을 여러 번 렌더한 `.state.json`이 있다.
- `topic_features.topic_type`, `event_type`, `angle_type`은 일부 부정확하다.
- 정책 적용 시 structured feature만 보지 말고 `topic_name`, `reason_summary`, headline, 기사 제목, 기사 수를 함께 본다.

## 과거 선택 패턴

### 팀 분포

editor report 기준 선택 topic의 팀 분포:

| 팀 | 선택 topic 수 |
| --- | ---: |
| KIA | 6 |
| 한화 | 5 |
| LG | 4 |
| 롯데 | 4 |
| 두산 | 3 |
| 삼성 | 3 |
| KT | 1 |
| NC | 1 |
| SSG | 1 |
| 키움 | 1 |

초기 정책:

- KIA, 한화, LG, 롯데는 계정 적합도 가산점을 준다.
- 두산, 삼성은 중간 가산점을 준다.
- KT, NC, SSG, 키움은 낮은 기본 가산점이지만, 사건성이나 기록성이 강하면 즉시 알림 대상이다.

### 소재 분포

`topic_features.topic_type` 기준 분포:

| 소재 | 선택 topic 수 |
| --- | ---: |
| injury | 11 |
| record | 9 |
| controversy | 3 |
| trade_or_roster | 3 |
| game_result | 1 |
| general_news | 1 |
| player_highlight | 1 |

단, 이 분류는 일부 오분류가 있다. 실제 주제명과 요약을 함께 보면 선택된 소재는 다음에 가깝다.

- 부상, 말소, 시즌 아웃, 핵심 선수 이탈
- 대기록, 홈런 선두, 통산 기록, 연속 기록
- 끝내기, 대승, 역전승, 연승/연패 탈출
- 복귀, 영입, 대체 외국인, 2군행, 엔트리 변동
- 징계 복귀, 사과, 논란, 감독/베테랑 발언
- 인기 선수의 활약 또는 거취

## 우선 알림할 주제

### 즉시 확인

`virality_potential_score`를 높게 준다.

- 핵심 선수 부상, 병원 이송, MRI, 수술, 시즌 아웃
- 1군 말소, 주전 이탈, 선발 로테이션 붕괴
- 징계, 사과, 논란, 감정적 반응이 예상되는 사건
- 끝내기 홈런, 역전승, 대승, 대형 승부처
- 통산 최다, 개인 500타점, 홈런 선두 같은 기록
- 인기 선수의 복귀, 복귀 무산, 거취 변화
- 대체 외국인 영입, 단기 계약, 방출/교체
- 인기 구단의 연승/연패, 순위 변화, 위기 또는 반등

예시:

- `문보경 발목 부상`
- `문동주 어깨 수술 확정 및 시즌 아웃`
- `삼성, 디아즈 끝내기 홈런`
- `롯데, 징계 복귀 선수 사과 기자회견`
- `고우석, LG 복귀 무산`

### 지켜볼 만함

`virality_potential_score`는 중간, `account_fit_score`가 높으면 알림한다.

- 인기 선수 활약
- 홈런, 세이브, 호투 등 개인 퍼포먼스
- 연승/연패 흐름
- 감독 코멘트
- 팬덤 반응이 예상되는 인터뷰
- 경기 결과 자체는 평범하지만 짧은 headline으로 잘 압축되는 소재

예시:

- `고우석 연속 세이브`
- `두산, 양석환 2군행`
- `KIA, 어린이날 대승 및 김도영 활약`
- `삼성, 구자욱 복귀 후 대승`

### 묶어서 확인

즉시 제작보다는 digest 후보로 둔다.

- 일반 경기 결과만 있는 기사
- 행사, 이벤트, 상품 출시
- 월간 MVP 후보 발표
- 리그 전체 관중 증가
- 특정 팀과 직접 팬덤 반응이 약한 일반 흥행 기사

예외:

- 행사/상품/리그 이슈라도 특정 인기 구단 팬덤과 강하게 연결되거나, 사진 소재가 좋고 headline이 강하면 알림할 수 있다.

## 낮은 우선순위 후보

다음은 기본적으로 낮게 본다.

- 기사 1개만 있고 후속 확산 가능성이 낮은 일반 인터뷰
- 경기 결과를 단순 요약한 기사
- 여러 팀을 넓게 다루지만 명확한 주인공이 없는 리그 기사
- 사진 후보가 거의 없고 visual hook이 약한 소재
- 이미 같은 날 비슷한 topic으로 알림 또는 제작한 소재
- 사실 확인이 더 필요한 루머성 거취 기사

단, false negative를 줄이는 것이 중요하므로 완전 제외하지 않는다. 낮은 우선순위는 digest로 보류한다.

## 사진 후보 정책

editor report 기준 선택 topic의 asset 수:

- 최소: 1
- 최대: 55
- 평균: 약 16.9

초기 기준:

- asset 10개 이상: 이미지 선택 여지가 충분하므로 가산점
- asset 4~9개: 제작 가능, 소재 강도에 따라 알림
- asset 1~3개: 사진 리스크 있음. 사건성이 강하면 알림하되 메시지에 사진 후보 부족을 표시

사진 후보가 적어도 선택된 사례가 있다.

- `KIA, 세븐일레븐과 협업 상품 출시`: asset 1
- `노시환 500타점 달성`: asset 2
- `한화 김경문 감독, 문현빈 루틴 지시 논란`: asset 4
- `두산, 양석환 2군행`: asset 4

따라서 사진 후보 수는 탈락 기준이 아니라 리스크 표시와 가산점 기준으로 사용한다.

## 제목 적합도

이 계정의 타이틀은 짧고 강한 단어로 압축되는 소재와 잘 맞는다.

좋은 형태:

- `문보경 부상`
- `시즌 아웃`
- `복귀 무산`
- `끝내기 홈런`
- `롯데 사과`
- `안우진 복귀승`
- `김도영 홈런 선두`
- `양석환 2군행`

가산점:

- 2~8자 내외의 강한 핵심어로 줄일 수 있음
- 팀명 + 사건어 조합이 자연스러움
- 선수명 + 결과어 조합이 자연스러움
- 팬이 댓글로 반응하기 쉬운 감정선이 있음

주의:

- 자동 생성 headline이 부정확할 수 있다.
- `topic_name`과 실제 기사 요약에서 더 좋은 headline 후보를 다시 뽑아야 한다.

## 점수 초안

### virality_potential_score

지금 터질 가능성이다.

가산 요소:

- 핵심 선수 부상/시즌 아웃/수술: +35
- 징계/논란/사과/감독 발언: +25
- 끝내기/역전승/대승/연승/연패 탈출: +20
- 대기록/개인 기록/리그 기록: +20
- 복귀/거취/영입/방출/말소: +20
- 기사 수 증가 또는 여러 매체 동시 보도: +10~20
- 발행 후 2시간 이내: +10
- 인기 구단 관련: +5~15

감점 요소:

- 단순 경기 결과: -15
- 일반 행사/상품/리그 흥행 기사: -10
- 기사 1개뿐이고 후속성 약함: -10
- 같은 주제로 이미 알림/제작됨: -30

### account_fit_score

이 계정의 기존 카드뉴스 스타일과 맞는 정도다.

가산 요소:

- KIA, 한화, LG, 롯데: +15
- 두산, 삼성: +10
- 부상/복귀/말소/거취/징계/논란: +20
- 기록/홈런/대승/끝내기: +15
- 짧은 headline으로 압축 가능: +15
- 사진 후보 10개 이상: +10
- 과거 제작 topic과 유사한 감정선: +10

감점 요소:

- visual hook 약함: -10
- headline이 설명형으로만 길어짐: -10
- 팀/선수 주인공이 불명확함: -10
- 계정 톤보다 리그 행정/비즈니스 성격이 강함: -10

## 알림 등급

### 즉시 확인

조건 예시:

- `virality_potential_score >= 70`
- 또는 핵심 선수 부상/시즌 아웃/징계/끝내기/대기록
- 또는 `account_fit_score >= 70`이면서 발행 후 3시간 이내

Discord 메시지에는 승인 CTA를 붙인다.

### 지켜볼 만함

조건 예시:

- `virality_potential_score >= 50`
- 또는 `account_fit_score >= 60`
- 기사량이 더 붙으면 즉시 확인으로 올릴 수 있는 후보

Discord 메시지에는 보류/승인 둘 다 가능하게 안내한다.

### 묶어서 확인

조건 예시:

- `virality_potential_score < 50`
- `account_fit_score < 60`
- 일반 경기 결과, 이벤트, 리그 이슈

즉시 알림 대신 일정 시간마다 digest로 묶는다.

## 중복 처리 규칙

같은 topic을 여러 번 알리지 않기 위해 다음을 비교한다.

- 대표 선수명
- 대표 팀명
- event keyword
- 기사 URL
- topic_name 유사도
- 발행 시간 window

중복으로 볼 가능성이 높은 예:

- `문동주 어깨 통증 말소`
- `문동주 어깨 수술 확정`

단, 상태가 크게 바뀌면 새 알림을 보낸다.

상태 변화 예:

- 통증 -> 말소
- 말소 -> 수술
- 부상 -> 시즌 아웃
- 복귀설 -> 복귀 무산
- 논란 -> 사과

## 거절 사유 수집

앞으로 승인하지 않는 후보에는 가능한 한 사유를 남긴다.

저장할 사유:

- `too_late`: 타이밍이 늦음
- `weak_topic`: 소재가 약함
- `duplicate`: 이미 다룸
- `no_good_image`: 쓸 사진이 없음
- `risky`: 사실관계나 표현 리스크 있음
- `not_account_fit`: 계정 톤과 안 맞음
- `manual_skip`: 기타 수동 스킵

거절 사유는 다음 ranker 개선에 사용한다.

## 초기 구현 메모

Phase 2~5에서 이 문서를 코드화할 때는 다음 순서가 좋다.

1. 위 가산점/감점 규칙을 단순 룰 기반 scorer로 구현한다.
2. structured `topic_features`는 참고값으로만 사용한다.
3. `topic_name`, `reason_summary`, 기사 제목에서 keyword를 직접 추출한다.
4. 점수와 함께 사람이 읽을 수 있는 추천 이유를 저장한다.
5. 낮은 점수 후보도 바로 버리지 말고 digest 후보로 저장한다.

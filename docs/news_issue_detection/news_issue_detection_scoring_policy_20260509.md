# 뉴스 이슈 감지 판단 기준 2026-05-09

이 문서는 fresh watcher가 새 기사를 보고 "Gemini 2차 판단에 올릴 이슈 후보인지" 판단하는 1차 기준을 정의한다.
Discord 알림은 이 기준을 통과한 뒤 Gemini가 최종 approve한 경우에만 보낸다.

핵심 원칙은 다음이다.

```text
많이 나온 기사보다, 지금 안 보면 늦는 변화를 우선 감지한다.
```

따라서 기준은 단순 기사량이 아니라 아래 세 가지를 중심으로 둔다.

```text
1. 상태 변화
2. 팬 반응 가능성
3. 카드뉴스 적합성
```

## 판단 목표

fresh watcher의 1차 rule이 판단해야 하는 질문:

```text
이 기사가 Gemini 2차 판단에 올릴 만큼 중요하고, 빠르게 반응할 가능성이 있는가?
```

Gemini 2차 판단에 올려야 하는 이슈:

```text
선수/팀/감독/외국인 관련 상태가 바뀜
팬들이 바로 반응할 만한 감정 포인트가 있음
짧은 제목과 3~5장 카드뉴스로 설명 가능함
최근 짧은 시간 안에 기사 확산 조짐이 있음
```

Gemini 2차 판단에 올리지 않아도 되는 기사:

```text
단순 경기 결과 종합
순위표/프리뷰/선발 예고
행사/이벤트/상품/중계 안내
맥락 없는 단순 인터뷰
2군/퓨처스 중심의 낮은 파급 기사
```

## 전체 점수 구조

점수는 0~100점 기준이다.

```text
issue_score =
  상태 변화 점수
+ 확산 신호 점수
+ 팬 반응 점수
+ 카드뉴스 적합성 점수
+ 계정 적합성 점수
+ 최근성 점수
- 저우선순위 감점
- 중복/낡은 이슈 감점
```

권장 초기 threshold:

```text
Gemini 2차 판단 대상: 65점 이상
즉시 확인 후보: 75점 이상
```

단, 부상/징계/논란처럼 강한 상태 변화가 있으면 65점 이상부터 `immediate`로 본다.
여기서 `immediate`는 Discord 알림 확정이 아니라 Gemini 검토 시 긴급 후보로 전달하는 값이다.

## 1. 상태 변화 점수


## 2. 확산 신호 점수

이슈가 실제로 퍼지고 있는지 판단한다.

### fresh article count

직전 collection window 안에서 새로 insert된 기사 기준이다.

```text
1건: +8
2건: +15
3건 이상: +22
```

단, A등급 상태 변화가 있으면 1건이어도 감점하지 않는다.

### source diversity

서로 다른 매체 수 기준이다.

```text
1개 매체: +0
2개 매체: +10
3개 이상: +18
```

### context growth

최근 24시간 관련 기사를 조회한 뒤, 시간대별 확산 흐름을 기준으로 본다.
1~3시간은 빠른 확산 여부를 보는 feature 중 하나이며, 조회 범위 자체는 24시간을 기본으로 한다.

```text
관련 기사 0~1건: +0
관련 기사 2건: +6
관련 기사 3~4건: +12
관련 기사 5건 이상: +18
```

관련 기사는 같은 팀, 선수 후보, 강한 키워드가 겹치는 기사다.

## 3. 팬 반응 점수

팬들이 댓글, 커뮤니티, SNS에서 반응할 만한 소재인지 본다.

### 강한 팬 반응: +20

```text
주전급 선수 부상/말소
외국인 선수 교체/방출/부진
감독 발언/판정/논란
징계/사과/사건
끝내기/대역전/대패
라이벌전 사건
연승/연패의 결정적 장면
```

### 중간 팬 반응: +12

```text
복귀전
콜업
대체 선발
신인 활약
기록 달성
보직 변경
```

### 약한 팬 반응: +5

```text
평범한 경기 후 인터뷰
일반 훈련 소식
시즌 전망
단순 기록 언급
```

## 4. 카드뉴스 적합성 점수

카드뉴스로 만들기 쉬운 주제인지 본다.

### 좋은 카드뉴스 후보: +18

조건:

```text
주인공이 명확함
한 문장 제목으로 압축 가능함
원인 -> 현재 상황 -> 향후 영향 구조가 가능함
사진/비주얼 소재가 있을 가능성이 높음
```

예:

```text
LG 오스틴 말소
KIA 김도영 부상 검진
한화 류현진 복귀전 확정
롯데 끝내기 승리
```

### 보통 카드뉴스 후보: +10

```text
주제는 분명하지만 맥락 설명이 조금 필요함
여러 선수가 함께 얽힘
제목 압축은 가능하지만 임팩트가 중간 정도임
```

### 낮은 카드뉴스 후보: -12

```text
주제가 너무 넓음
한 문장 제목으로 압축하기 어려움
정보가 흩어져 있음
단순 종합 기사임
```

예:

```text
오늘 프로야구 5경기 종합
주말 3연전 관전 포인트
10개 구단 선발 투수 예고
혼전의 순위 싸움
```

## 5. 계정 적합성 점수

팀 주체가 불명확하면 감점한다.

## 6. 최근성 점수

fresh watcher는 빠른 반응이 목적이므로 최근성을 반영한다.

대표 기사 발행 시각 기준:

```text
10분 이내: +10
30분 이내: +6
60분 이내: +3
60분 초과: +0
```

발행 시각이 없으면 수집 시각을 fallback으로 사용한다.

## 8. 중복/낡은 이슈 감점

이미 보낸 이슈는 다시 보내지 않는다.

### 중복 처리

최근 72시간 automation job과 비교한다.

중복 기준:

```text
같은 대표 기사 URL
기사 URL overlap
같은 topic_fingerprint
같은 article_url_fingerprint
같은 팀 + 같은 선수 + 같은 강한 키워드
```

중복이면:

```text
-100
```

즉 job 생성하지 않는다.

### 낡은 이슈

대표 기사 기준 발행 3시간 이상 지났고 fresh article이 1건뿐이면 감점한다.

```text
-20
```

## notification level 결정

notification level은 로컬 rule의 추천값이다.
최종 Discord 알림 여부와 레벨은 Gemini 2차 판단 결과로 확정한다.

### immediate

조건 중 하나를 만족하면 `immediate`.

```text
issue_score >= 75
A등급 상태 변화가 있고 issue_score >= 65
논란/징계/사건 키워드가 있고 issue_score >= 65
주전급 선수 부상/말소로 판단되고 issue_score >= 65
```

### watch

```text
65 <= issue_score < immediate 기준
```

### 생성하지 않음

```text
issue_score < 65
중복 이슈
행사/홍보성만 있는 기사
단순 종합/프리뷰 기사
```

## 운영 튜닝 기준

아래 값은 고정 정책이 아니라 초기 운영값이다.
승인/거절, 제작 여부, 제작 후 반응, 오탐, 누락 이슈 데이터를 쌓으면서 rule 점수와 Gemini prompt를 같이 보정한다.

### 알림이 너무 많을 때

조정 순서:

```text
1. min_issue_score 65 -> 70
2. 단일 기사 + 단일 매체 penalty 강화
3. low priority 감점 강화
4. immediate 기준 75 -> 80
```

### 알림이 너무 적을 때

조정 순서:

```text
1. min_issue_score 70 -> 65
2. A등급 상태 변화는 기사 1건이어도 허용
3. 인기 구단 team_fit 보정 확대
4. context 조회는 24시간을 유지하되 관련도 필터를 완화
```

### 오탐이 많을 때

주로 강화할 기준:

```text
단순 종합/프리뷰 감점
행사/홍보 감점
2군/퓨처스 감점
약한 인터뷰성 감점
중복 fingerprint
```

### 놓치는 이슈가 많을 때

주로 완화할 기준:

```text
상태 변화 키워드 확장
선수명 추출 개선
24시간 context 내 관련도 필터 완화
source diversity 의존도 축소
```

## 최종 판단 우선순위

코드 구현에서 애매한 경우 아래 우선순위를 따른다.

```text
1. 상태 변화가 있는가?
2. 주체가 명확한가?
3. 팬들이 반응할 감정 포인트가 있는가?
4. 카드뉴스 제목으로 압축 가능한가?
5. 최근 짧은 시간에 퍼지고 있는가?
6. 계정에서 반응이 잘 나올 구단인가?
7. 단순 종합/프리뷰/홍보성은 아닌가?
8. 최근에 이미 보낸 이슈는 아닌가?
```

가장 중요한 결론:

```text
기사량은 보조 신호다.
핵심 신호는 상태 변화다.
로컬 점수는 최종 알림 판단이 아니라 Gemini 2차 판단 대상을 고르는 1차 필터다.
판단 기준은 운영 데이터에 따라 계속 수정한다.
```
